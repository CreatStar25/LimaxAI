## Контекст: зачем переписывать старые промпты

Старые промпты часто **перегружают процесс** — раньше моделям нужны были жёсткие рельсы. С GPT-5.5 это шум: сужает пространство поиска и даёт механические ответы. **Перепишите с нуля**, не копируйте старые шаблоны.

Сооснователь Django Simon Willison отметил, что OpenAI рекомендует **переписать с нуля**, а не полагать, что старые промпты работают. При переходе на GPT-5.5 (или в LimaxAI) относитесь к промптам как к новому продукту.

## Изменение 1: от пошаговости к результату

**Старый стиль (процесс):**

> Проверь A, затем B, сравни поля по одному, обработай исключения, выбери инструмент и объясни пользователю весь процесс.

**Новый стиль (результат):**

> Полностью решите запрос пользователя. Опирайтесь на политику и данные аккаунта; сообщите, что сделано и что осталось. Один критический вопрос при нехватке данных.

**Опишите пункт назначения, а не каждый поворот.** GPT-5.5 сам планирует путь; жёсткий сценарий мешает лучшим решениям.

## Изменение 2: меньше «обязательно» и «запрещено»

GPT-5.5 чувствителен к абсолютным формулировкам; противоречивые «должен» ломают оба правила.

| Ситуация | Стиль |
| --- | --- |
| Безопасность, обязательные поля | всегда/никогда |
| Искать или спрашивать | условия |

Пример: вместо «перед ответом обязательно искать» — «если контекста достаточно, отвечай без поиска».

## Изменение 3: роль снова нужна, но коротко

**Role** — 1–2 предложения о функции; **Personality** — тон и стиль совместной работы, не резюме на 20 лет.

- **Steady:** reliable collaborator—calm, direct; advance without unnecessary check-ins.
- **Expressive:** conversational, curious; ask when ambiguous, decide when context is enough.

## Изменение 4: преамбула перед многошаговой работой

Перед инструментами отправьте 1–2 видимых предложения: запрос принят, первый шаг — …

Продукты вроде Codex уже используют этот шаблон (напр. «Анализирую кодовую базу, начну с тестов») для API и длинных задач ChatGPT.

## Изменение 5: явные условия остановки

Фраза «найди информацию и ответь» без стопа ведёт к циклам поиска.

Две стратегии:

1. **Бюджет поиска:** по умолчанию один раунд; отвечайте, если хватает первой страницы. Второй раунд — только если на первой нет ответа, нет ключевых данных или нужна полнота.
2. **Стоп цикла:** после каждого инструмента спросите, хватает ли доказательств на главный вопрос; если да — стоп.

## Изменение 6: скелет из 7 блоков

| Блок | Назначение |
| --- | --- |
| Role | Функция |
| Personality | Тон |
| Goal | Результат |
| Success | Готово |
| Constraints | Границы |
| Output | Формат |
| Stop | Когда остановиться |

Для простых вопросов достаточно Goal + Output. OpenAI developer console also offers **Optimize** to reshape prompts to this structure.

## Изменение 7: формат и краткость

GPT-5.5 по умолчанию краток. В API — `text.verbosity` (по умолчанию `medium`, для короче — `low`) и **Structured Outputs**.

Пример оформления:

> Естественные абзацы по умолчанию; заголовки и списки — только для ясности.

При переписывании сначала **что сохранить**:

> Сохраняйте структуру, длину и голос; улучшайте ясность — без новых тезисов без запроса.

Без правил сохранения модель часто расширяет текст.

## Миграция: reasoning с низкого уровня

Пересмотрите `reasoning.effort`: по умолчанию medium, часто хватает low. Повышайте только по метрикам качества.
